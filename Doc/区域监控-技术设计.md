研判面板为独立 feature，贴右侧或浮层；通过 Store 订阅 `activeSegment`、`selectedTargetId`、`zoneEvents`。

### 3.2 坐标系

- 归一化平面坐标 `x, y ∈ [0, 1]`
- 核心中点 `center = { x: 0.5, y: 0.5 }`
- 圈层半径（归一化）：预警 `0.48`、跟踪 `0.28`、反制 `0.12`

```typescript
function detectZone(pos: Point, center: Point, zones: ZoneRadii): ZoneLevel {
  const d = Math.hypot(pos.x - center.x, pos.y - center.y);
  if (d <= zones.counter) return 'counter';
  if (d <= zones.track) return 'track';
  if (d <= zones.warning) return 'warning';
  return 'outside';
}
```

威胁色映射：


| zone / 角色        | threatLevel | 颜色  |
| ---------------- | ----------- | --- |
| 己方               | friendly    | 绿   |
| 鸟群               | none        | 灰   |
| outside + 不明     | normal      | 蓝   |
| warning 圈 / 预警目标 | warning     | 黄   |
| counter / 警报     | alert       | 红   |
| 已处置              | disposed    | 灰   |


### 3.3 运动模型

#### 3.3.1 `polygonLoop` — 非预警背景目标

沿**不规则闭合多边形**匀速循环移动，永不停止。

```json
{
  "type": "polygonLoop",
  "vertices": [
    { "x": 0.62, "y": 0.38 },
    { "x": 0.71, "y": 0.52 },
    { "x": 0.58, "y": 0.68 },
    { "x": 0.45, "y": 0.55 }
  ],
  "lapDurationSec": 120
}
```

算法要点：

1. 计算各边长度之和 `perimeter`
2. `speed = perimeter / (lapDurationSec / speedMultiplier)`
3. `elapsed` 对 `lapDurationSec / speedMultiplier` 取模得 `lapElapsed`
4. 沿边插值得到当前坐标

适用：**B-101**、**U-001**、**U-005**（各用不同 `vertices` / `lapDurationSec`，避免轨迹重合）。

#### 3.3.2 `linear` — 分段演示目标

沿**直线**从 `from` 飞向 `to`，**到达核心中点的用时**由配置指定。

```json
{
  "type": "linear",
  "from": { "x": 0.82, "y": 0.62 },
  "to": { "x": 0.38, "y": 0.44 },
  "toCoreSec": 30,
  "altitude": { "from": 120, "to": 80 },
  "speedKmh": 32,
  "headingText": "逼近核心区域",
  "onComplete": "hover"
}
```

**线速度计算**（保证飞过「起点→核心中点」段的时间 = `toCoreSec / speedMultiplier`）：

```typescript
function linearSpeed(from: Point, to: Point, center: Point, toCoreSec: number, multiplier: number) {
  const coreFrac = segmentRatioToCenter(from, to, center); // 0~1，核心在中点上的投影比例
  const totalLen = dist(from, to);
  const lenToCore = totalLen * coreFrac;
  const effectiveSec = toCoreSec / multiplier;
  return lenToCore / effectiveSec; // 归一化单位/秒
}
```

整段飞行总时长：`totalLen / linearSpeed`。

**完成后**：`onComplete: "hover"` → 停在 `to` 点，不再移动。

### 3.4 演示状态机（支持并行）

```
                         ┌──────┐
                         │ idle │
                         └──┬───┘
            POST start (按 segment，可多次、可并行)
                            ▼
         ┌──────────────────────────────────────┐
         │  running（多目标各自独立计时）         │
         │  U-004 / U-002 / U-003 可同时在飞     │
         └──────────────┬───────────────────────┘
                        │ POST reset
                        ▼
         ┌──────────────────────────────────────┐
         │  idle（全部演示目标隐藏、计时清零）      │
         └──────────────────────────────────────┘
```

规则：

1. **各分段独立**：`POST /demo/start { segment }` 只启动对应目标，各自记录 `segmentStartAt[targetId]`
2. **可同时触发**：三个按钮或「全部开始」可连续点击；U-004（30s）→ U-002（45s）→ U-003（60s）先后抵近核心
3. 某目标首次进入 `warning` 圈 → 若研判面板未锁定其他目标，可自动选中；**选中态以用户点击为准**
4. 进入 `track` 圈 → 该目标在研判面板中「一键下发」可点击（按 `effectiveSegment` 规则）
5. 直线结束 → `hovering`，停在终点
6. `POST /demo/reset` → 全部演示目标 `visible=false`，清空各 `segmentStartAt`

Store 结构示意：

```typescript
demo: {
  speedMultiplier: number;
  segmentStartAt: Record<string, number>; // targetId → epoch ms
  runningTargets: Set<string>;          // 正在飞的演示目标
}
```

### 3.5 全局速度倍率

配置来源：`scene.demoTiming` + 运行时 UI 调节。

```json
{
  "demoTiming": {
    "alertToCoreSec": 30,
    "unknownToCoreSec": 45,
    "warningToCoreSec": 60,
    "speedMultiplier": 1.0,
    "presets": [0.5, 1.0, 1.5, 2.0]
  }
}
```

UI：下拉或按钮组切换 `speedMultiplier` → `POST /api/v1/monitoring/demo/timing`。

**调速不跳变**：调整时记录 `adjustAt` 与当时各目标的 `snapshotPosition`，以新倍率重算 `segmentStartAt`：

```typescript
// 伪代码
const progress = computeProgress(oldStartAt, oldMultiplier);
segmentStartAt = now - progress * effectiveDuration(newMultiplier);
```

### 3.6 行为时间线（自动生成）

直线演示过程中，**首次**触发 zone 变化时写入 `behaviorTimeline`（供研判展示）：


| 事件        | 触发条件                           |
| --------- | ------------------------------ |
| 雷达 A 首次发现 | 首次 `zone !== outside`          |
| 进入跟踪区     | `zone` 变为 `track`              |
| 进入反制区     | `zone` 变为 `counter`            |
| 加速逼近核心区域  | 进入 `counter` 且 `speedKmh` 高于阈值 |


时间戳 = `状态栏当前本地时间 - offset`，保证早于「现在」。

### 3.7 航迹与 PPI

- **航迹**：仅当前分段目标；勾选后缓存最近 120 个位置点；`disposed` 清空
- **PPI**：CSS 1 秒一圈，与运动倍率无关

### 3.8 与预警研判联动


| 事件                     | 载荷                                          |
| ---------------------- | ------------------------------------------- |
| `segment:started`      | `{ segment, targetId }`                     |
| `target:zone-changed`  | `{ targetId, zone, threatLevel }`           |
| `target:arrived`       | `{ targetId }` 直线到达悬停                       |
| `target:manual-tagged` | `{ targetId, tag, effectiveSegment }`       |
| `judgment:dispatch`    | 研判 → 监控，目标 `dispositionStatus: in_progress` |
| `disposition:complete` | 目标 `disposed`，停止运动                          |


#### 研判模板路由 `effectiveSegment`

每个目标在 Store 中维护 `effectiveSegment`，决定加载哪份 Mock 研判与下发规则：


| 目标    | 初始 `effectiveSegment` | 说明         |
| ----- | --------------------- | ---------- |
| U-004 | `alert`               | 固定警报流      |
| U-002 | `unknown`             | 不明模板；标记后切换 |
| U-003 | `warning`             | 固定预警流      |


**U-002 标记切换**（`POST /targets/U-002/manual-tag`）：

```typescript
function applyManualTag(targetId: string, tag: ManualTag) {
  target.manualOverride = tag;
  if (tag === 'alert') target.effectiveSegment = 'alert';
  else if (tag === 'warning') target.effectiveSegment = 'warning';
  else target.effectiveSegment = 'unknown'; // friendly / normal
  reloadJudgment(target.effectiveSegment);
  emit('target:manual-tagged', { targetId, tag, effectiveSegment: target.effectiveSegment });
}
```

标记为 `alert` / `warning` 后：地图颜色更新、研判面板切换模板、进入 `track` 圈后可走**对应 segment 的下发与处置闭环**。

研判静态文案：`GET /api/v1/judgment/{effectiveSegment}.json`。

## 6. MotionEngine 伪代码

```typescript
function tick(now: number, store: MonitoringStore) {
  const { speedMultiplier, segmentStartAt } = store.demo;

  for (const target of store.targets) {
    if (!target.visible || target.motionState === 'disposed') continue;

    if (target.motion.type === 'polygonLoop') {
      target.position = evalPolygonLoop(target.motion, now, speedMultiplier);
    }

    if (target.motion.type === 'linear' && segmentStartAt[target.targetId]) {
      const elapsed = (now - segmentStartAt[target.targetId]) / 1000;
      const { position, done } = evalLinear(target.motion, elapsed, speedMultiplier, scene.center);
      target.position = position;
      if (done) target.motionState = 'hovering';
    }

    const zone = detectZone(target.position, scene.center, scene.zones);
    if (zone !== target.zone) emitZoneChanged(target, zone);
    target.threatLevel = resolveThreat(target, zone); // manualOverride 优先
  }
}
```

## 7. 演示操作流程（联调剧本）

1. 进入 `/realtime`，背景目标 B-101 / U-001 / U-005 沿多边形循环
